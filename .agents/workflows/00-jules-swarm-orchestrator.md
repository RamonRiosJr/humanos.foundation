## Swarm Topology (The "Crazy" Blueprint)

When deployed, the Jules Swarm spins up three independent, adversarial instance nodes that read the same files but have entirely different directives.

### Node 1: "The Inquisitor" (Violation Tracer)
*   **Directive:** Trace every single sentence in the ecosystem for FDA SaMD and FTC HBNR violations.
*   **Behavior:** It does not care about grammar, logic, or business flow. It only looks for words like "diagnose," "triage," "recommend," or "HIPAA-compliant" and flags them as catastrophic legal risks. It generates a `violation_trace.json`.

### Node 2: "The Contradiction Hunter" (Logic Engine)
*   **Directive:** Hunt for paradoxes across the ecosystem. 
*   **Behavior:** It compares files against each other. If `04-Business-Model.md` says we are targeting pediatric clinics, but `Pitch_Investor.md` says we are targeting nursing homes, it flags a "Strategic Contradiction." If the `TODO.md` says we are broke, but a `JD` file offers a $120k base, it flags a "Chronological Contradiction."

### Node 3: "The Supreme Arbitrator" (Resolution Node)
*   **Directive:** Consume the `.json` outputs from Node 1 and Node 2.
*   **Behavior:** It forcefully resolves the conflicts, rewrites the offending lines in the source Markdown files, and outputs a `SWARM_RESOLUTION.md` report to the Architect.
